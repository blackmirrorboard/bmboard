# 06. できること・できないこと一覧

> 「これって魔法で実現できるの？」に即答するための対応表。
> **このページの存在意義：失敗する前に境界を知ることで、作る側のストレスを最小化する。**

---

## 🟢 できること（確実に動く）

### 描画オブジェクトの操作

| やりたいこと | 方法 |
|---|---|
| 図形を新規作成 | `BM.create(type, props)` — type は5種のみ |
| 既存の図形を変形 | `BM.update(obj, {...})` |
| 図形を移動 | `BM.translate(obj, dx, dy)` |
| 線の色を変更 | `BM.setStroke(obj, '#RRGGBB')` |
| 塗りの色を変更 | `BM.setFill(obj, '#RRGGBB')` |
| 図形を削除 | `BM.remove(obj)` |
| 全削除 | `BM.clear()` |
| 選択中を取得 | `BM.getSelected()` |
| 全オブジェクト取得 | `BM.all()` |
| 条件で絞り込む | `BM.find(o => o.type === 'circle')` |

### ランダム・乱数処理

| やりたいこと | 方法 |
|---|---|
| 範囲指定の乱数 | `BM.rand(min, max)` |
| ランダム散布 | `BM.rand` を座標指定に使う |
| ランダム色 | `'#' + Math.floor(Math.random()*0xFFFFFF).toString(16)` |

### テーマ操作

| やりたいこと | 方法 |
|---|---|
| 現在のテーマ取得 | `BM.getMode()` — `'dark'` / `'light'` / `'gray'` |
| テーマ切替 | `BM.setMode('dark')` |

### ターミナル出力

| やりたいこと | 方法 |
|---|---|
| ログ出力（成功） | `BM.log('done')` または `BM.log('ok', 't-ok')` |
| ログ出力（エラー） | `BM.log('failed', 't-err')` |
| ログ出力（薄文字） | `BM.log('info', 't-dim')` |

### 非同期処理（`fetch-img` 的なパターン）

| やりたいこと | 方法 |
|---|---|
| 画像を読み込んでから配置 | `new Image()` + `img.onload = () => {...}` |
| 外部 API からデータ取得 | `fetch(url).then(r => r.json()).then(...)` — CORS 制限あり |
| 一定時間後に何かする | `setTimeout(() => {...}, 1000)` |

---

## 🟡 できるが注意が必要

### 外部 API への通信

| やりたいこと | 注意点 |
|---|---|
| 画像の取得 | **CORS ヘッダーがないサーバーはブラウザがブロック** |
| JSON API 呼び出し | 同上。`Access-Control-Allow-Origin: *` 必須 |
| 有名な使える API | Lorem Flickr / JSONPlaceholder / 公開 RSS feed |
| 使えない API | OpenSky（自社ドメイン限定）/ Twitter API（認証必須）等 |

### Canvas 上の画像加工

| やりたいこと | 注意点 |
|---|---|
| 画像にフィルタをかける（`$ monoclo`） | `crossOrigin='anonymous'` 付きで読み込めばOK |
| dataURL 化（永続化） | CORS が通っていない画像は `toDataURL()` でエラー |
| 失敗時のフォールバック | 元 URL をそのまま使う（エクスポートバンドルには入らない） |

### 大量オブジェクト生成

| やりたいこと | 注意点 |
|---|---|
| 1000個以上の図形を一度に作成 | 動くがパフォーマンス低下 |
| 10000個以上 | ブラウザがフリーズする可能性。ループ外で `BM.redraw()` を1回だけ呼ぶ |
| 目安 | 500〜1000個が快適に動く範囲 |

### 長時間実行する魔法

| やりたいこと | 注意点 |
|---|---|
| アニメーション（setInterval） | 動くが停止できる仕組みを自分で作る必要あり |
| 永続的にループする処理 | ブラウザが重くなる。終了条件を明示する |

## 🔴 できないこと（やってもエラー／効果なし）

### BM.create の制限

| ❌ 無効 | ✅ 代わりに使う |
|---|---|
| `BM.create('rect', ...)` | `BM.create('square', ...)` |
| `BM.create('ellipse', ...)` | `BM.create('circle', ...)`（rx ≠ ry で楕円化） |
| `BM.create('line', ...)` | `BM.create('arrow', ...)` |
| `BM.create('polygon', ...)` | （未対応。複数図形で近似） |
| `BM.create('path', ...)` | （未対応。ペンで手描き） |
| `BM.create('image', {src:...})` | （`BM.create` 経由の画像作成は非推奨。`BM.all().push({type:'image',...})` で直接追加） |

### JavaScript の機能制限

| ❌ できない | 理由 |
|---|---|
| `import / require` | モジュール機能は無効 |
| `async/await`（action 全体で） | action は同期関数として実行 |
| 外部ライブラリ（jQuery, Lodash, React...）| ロードされていない |
| ブラウザ外への通信 | サーバー無し。すべてローカル |
| ファイルシステム直接アクセス | `fs.readFile` 等は使えない |
| Node.js 固有機能 | ブラウザ環境 |

### DOM 操作

| できる／できない | 内容 |
|---|---|
| ✅ `document.createElement` | 使えるが推奨されない（魔法の範囲外） |
| ✅ `new Image()` | 画像取得には必須 |
| ⚠️ `document.getElementById` | 使えるが、内部要素に依存すると脆い |
| ❌ ページ全体の書き換え | 意図せずボード自体を壊す可能性 |

### 永続化・保存の制限

| できる／できない | 内容 |
|---|---|
| ✅ IndexedDB 経由の自動保存 | `BM.save()` で Undo 履歴に残る |
| ✅ プロジェクト保存（`save`/`load`） | ユーザー操作で明示的に |
| ❌ 別ドメインのストレージに書き込み | セキュリティ上不可 |
| ❌ サーバーへのアップロード | サーバー無し |

## 📋 オブジェクト型の完全対応表

`BM.all()` で返ってくるオブジェクトの型と、できる操作：

| type | 読める | 書ける | 移動 | 色変更 | 削除 |
|---|---|---|---|---|---|
| `circle` | ✅ | ✅（create）| ✅ | ✅ | ✅ |
| `square` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `triangle` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `arrow` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `text` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `stroke`（ペン描画） | ✅ | ❌（ユーザー描画のみ） | ✅ | ✅（v1.2 修正） | ✅ |
| `image` | ✅ | ❌（`fetch-img` か `I` キー経由） | ✅ | — | ✅ |

### オブジェクトの data プロパティ一覧

`obj.data` で読み書きできるプロパティ：

```
circle:   { cx, cy, rx, ry, rotation, fill, stroke, strokeOff }
square:   { x, y, w, h, rotation, fill, stroke, strokeOff }
triangle: { x, y, w, h, rotation, fill, stroke, strokeOff }
arrow:    { x1, y1, x2, y2, rotation }
text:     { x, y, text, fontSize, rotation, fill, stroke }
stroke:   { pts: [{x,y}, ...], lw, closed, fill, stroke, strokeOff }
image:    { x, y, w, h, src (dataURL or URL) }
```

## 💡 よくあるユースケースと実現可能性

| 要件 | 可能？ | 備考 |
|---|---|---|
| 選択中を複製 | ✅ | `BM.create` で同じ props を再作成 |
| グリッド配置 | ✅ | `BM.create` をループで |
| 色をランダム化 | ✅ | `BM.setStroke` + ランダム色 |
| 画像をキーワードで取得 | ✅ | Lorem Flickr 経由（CORS OK） |
| フィルムグレイン質感 | ✅ | 微小 square を散布 |
| オブジェクトの回転 | ✅ | `obj.data.rotation = ラジアン値` |
| オブジェクトのアニメーション | ⚠️ | `setInterval` で可。終了条件必須 |
| 音を鳴らす | ⚠️ | `new Audio().play()` で可能だがユーザー操作必須 |
| PDF として書き出し | ❌ | 機能無し（PNG/SVG なら可） |
| GIF アニメ書き出し | ❌ | 機能無し |
| SVG として書き出し | ✅ | `$ svg` で対応（v1.2 新規・Illustrator/Figma 互換） |
| 動画埋込 | ❌ | `video` タグを使う image 型が未対応 |
| カメラでセルフィー表示 | ✅ | `Black Mirror` で右上ワイプ（v1.2 追加・Secure Context 必要）|
| URL でキャンバス共有 | ✅ | `share` で URL 化、別ユーザーが開くと同じ画面（v1.2 追加）|
| リアルタイム共同編集 | ❌ | サーバー無し設計（URL 共有は非同期スナップショット扱い） |
| クラウド同期 | ❌ | ローカル専用（`export data` で手動バックアップ） |

## 🧭 迷ったときのフローチャート

```
やりたいこと
  │
  ├─ 既存オブジェクトを変える？
  │     └→ BM.getSelected / BM.all / BM.all で取得し、
  │         BM.update / BM.setStroke / BM.setFill / BM.translate で変更
  │
  ├─ 新しく作る？
  │     └→ BM.create(type, props)  ※ type は5種のみ
  │
  ├─ 画像を扱う？
  │     └→ fetch-img（既存魔法）を参考にする
  │
  ├─ 外部 API からデータ取得？
  │     └→ CORS が通っているか確認してから fetch()
  │
  └─ それ以外
        └→ このページの「🔴 できないこと」に該当するかチェック
```
