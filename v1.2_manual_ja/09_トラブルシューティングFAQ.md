# 09. トラブルシューティング FAQ

> よくある問題と解決方法。まずはここをチェック。

---

## 描画関連

### Q. 絵が描けない

- ツールが「セレクトツール（S）」になっていませんか？ → `P` キーでペンに切替
- モバイルでブラウザのジェスチャに吸われていませんか？ → 2本指操作を避けて1本指で描く

### Q. 選択しているのに色が変わらない

v1.2 以前は **ペンで描いた線の色変更バグ** がありました。2026-04-19 に修正済み。最新版の app.html を使っているか確認してください。

### Q. 描いた絵が消えた

- ブラウザのデータが消えた可能性
- シークレットモード／プライベートブラウズ中に作業すると保存されない
- 対策：作業途中でも `save <名前>` か `export data` で定期的にバックアップ

## ターミナル関連

### Q. コマンドが認識されない（`command not found`）

- スペルミスを確認
- `list` で魔法一覧を見て、意図した名前と一致しているか確認
- 大文字／小文字は関係ない（自動で小文字化される）
- `$` が複数ついていないか（`$$scatter` 等）

### Q. `$ biwako-blue` などが「実行した」とログに出るのに何も起きない

- 対象オブジェクトが **0個** の可能性 → 何か描いてから実行
- スクロール／ズームで対象が画面外にいる可能性 → Space+ドラッグで探す
- v1.2 以前のペン線色バグ → 最新版に更新

### Q. ターミナルに引数を打っても消える（例：`fetch-img sea` が `fetch-img` としか表示されない）

v1.2 以前の**表示バグ**。2026-04-19 に修正済み。引数は正しく処理されていたが、表示だけ欠落していた。最新版を使えば直っています。

## 魔法関連

### Q. `$ fetch-img` で画像が出ない

原因チェックリスト：

1. `list` で `[preset] $fetch-img` になっているか確認
2. `[ user ] $fetch-img` になっていたら `forget fetch-img` → ブラウザリロード
3. DevTools の Network タブで Lorem Flickr が 503 を返していないか
4. オフライン環境では動かない（外部 API 依存）

### Q. 自作魔法を登録したのにエラーが出る

よくあるエラーと対処：

| エラー | 原因 | 対処 |
|---|---|---|
| `unknown type "rect"` | `BM.create('rect', ...)` を使った | `square` に変更 |
| `BM.create is not a function` | action の書き方ミス | `BM.create(type, props)` の引数確認 |
| `Cannot read property 'x' of undefined` | 存在しないプロパティアクセス | `BM.getById` 結果の null チェック |
| `Unexpected token` | JSON 構文エラー | カンマ忘れ・引用符不一致 |

### Q. 取り込んだ魔法が `[ user ] $grain_2026-04-19` になっている

**リネームは仕様です**。同名の既存魔法を保護するため。

- 元の `$grain` を消したいなら `forget grain`
- リネーム版をメインで使いたいなら `alias grain=grain_2026-04-19`

### Q. preset を削除したい

**不可能**。`forget scatter` 等はエラー：`[scatter] is a preset`。
preset は上書き・削除できない仕様。自作魔法だけが削除可能。

### Q. AI に頼んだ魔法が動かない

- [[06_できることできないこと一覧|06. できる・できない]] を AI に渡して再試行
- エラーメッセージをそのままコピーして AI に返す
- 特に `rect` → `square` / `async/await` → Promise チェーン に書き換えさせる

## 書き出し・取り込み関連

### Q. `export` を打ったのに PNG がダウンロードされない

- 選択中のオブジェクトが無く、キャンバスも空 → 何か描いてから実行
- ブラウザのダウンロード制限で止まっている → ブラウザ設定を確認
- モバイル Safari は共有シートが開く（直接ダウンロードではない）

### Q. D&D したのに何も起こらない

- ファイル拡張子が `.json` でない → リネームしてから再試行
- JSON の中身が壊れている → テキストエディタで開いて JSON 構文チェック
- 複数ファイルをドロップした → 最初の1件しか処理されない

### Q. バンドルを D&D したら絵が消えた

バンドルの読み込みダイアログで **OVERWRITE** を選ぶと現キャンバスが消えます。
復旧：`Cmd+Z`（Undo）で戻せる場合あり。できなければ最後の `save` で保存した状態を `load` で復元。

## パフォーマンス関連

### Q. 動作が重い

原因候補：

1. **オブジェクトが多すぎる** → `objects` コマンドで数を確認。1000個超えるなら分割推奨
2. **画像が多い** → dataURL 埋込の画像が多いとメモリを食う
3. **ブラウザが古い** → Chrome / Safari を最新に
4. **モバイルで大規模ボード** → PC 推奨

### Q. `fetch-img` で大量（20枚以上）取ると固まる

Lorem Flickr のレート制限にかかっている可能性。10枚ずつ取得するのを推奨。

## 保存関連

### Q. ブラウザを閉じたら全部消えた

- 通常はブラウザの IndexedDB に保存されているはず
- シークレットモードでは保存されない
- ブラウザのサイトデータ削除で消える
- 対策：`save <名前>` + `export data` で外部ファイル保存

### Q. 他のマシンで同じ状態にしたい

サーバー同期は **非対応**。以下の手順で手動同期：

```
元マシン:  export data → USER ONLY チェック → ファイル保存
↓
ファイル転送（iCloud / Drive / USB 等）
↓
別マシン:  そのファイルを D&D → OVERWRITE 選択
```

v1.2 以前のバグ。2026-04-18 修正済み（入力欄の font-size を 16px 以上に調整）。最新版では発生しません。

### Q. 長押しペーストで魔法が登録されない

v1.2 以前のバグ。2026-04-18 修正済み（setTimeout フォールバック追加）。

### Q. タッチで描画できない

- Safari の設定で JavaScript が無効になっていないか確認
- 別ブラウザ（Chrome for iOS）で試す

## AI 活用時の問題

### Q. AI が `rect` を使う魔法を作ってくる

**よくあるミス**。以下のように AI に伝える：

```
rect はこの環境にありません。代わりに square を使ってください。
BM.create('square', {x:0, y:0, w:100, h:100}) の形式です。
```

### Q. AI が async/await を使ってくる

action は同期関数として実行されるため動きません。以下を指示：

```
async/await は使わず、.then() のプロミスチェーンで書いてください。
```

### Q. AI の魔法が読み込めない

JSON のシンタックスを確認：

- ダブルクォートで囲まれているか
- 中に `"` が含まれる場合は `\"` でエスケープされているか
- 改行が `\n` でエスケープされているか（または action を1行にまとめる）
