# 08. 魔法の取り込み

> 他人の魔法・ファイルを自分のボードに取り込む方法の完全ガイド。

---

## 取り込みの3つの経路

| 経路 | 対応形式 | 使う場面 |
|---|---|---|
| ターミナルにペースト | JSON テキスト全形式 | SNS からコピペ |
| ボードにファイル D&D | `.json` ファイル | ダウンロードしたファイル |
| `register` コマンド | 手書きの JS コード | 自分で一行で書くとき |

---

## 1. ペーストでの取り込み

### 使い方

1. SNS や Discord から JSON をコピー
2. Black Mirror Board のターミナル入力欄にペースト（Cmd+V）
3. **Enter すら押さなくても自動登録**される

### サポートされる JSON 形式（4種）

| 形式 | 例 |
|---|---|
| 単一魔法 | `{"command":"grain","action":"..."}` |
| 魔法の配列 | `[{"command":"a",...},{"command":"b",...}]` |
| 魔法パッケージ | `{"format":"blackmirror.spells","commands":[...]}` |
| 完全バンドル | `{"format":"blackmirror.bundle","project":...,"commands":[...]}` |

### 成功ログ

```
paste > [JSON payload intercepted]
→ schema check... ✓
→ binding $grain...
[OK] spell acquired — run: $grain
```

### モバイル（iOS）でのペースト

長押しで「ペースト」→ 自動登録。v1.2 で互換性問題は解消済み。

---

## 2. ファイル D&D での取り込み

### 使い方

1. デスクトップの JSON ファイルをつかむ
2. Black Mirror Board の画面上にドラッグ
3. 画面全体に **点線枠「◼ DROP JSON HERE」** が表示される
4. ドロップ → 自動判別＆登録

### 受付条件

- ファイル拡張子 `.json` または MIME `application/json`
- 複数ファイルを同時にドロップしても、**最初の1ファイルのみ** 処理
- それ以外はエラー：`[drop] "xxx" is not a JSON file.`

### 成功ログ（魔法パッケージの場合）

```
drop > [BM_SPELLS_20260419.json]
→ schema check... ✓
→ 3 spell(s) in payload
  [OK]     $my-scatter
  [RENAME] $grid → $grid_2026-04-19
  [SKIP]   $paint-red — identical
— 1 added · 1 renamed · 1 skipped
```

## 3. 衝突時の自動リネーム

**同じ名前の魔法が既に存在する場合、日付サフィックスで自動リネーム**される。preset も絶対に上書きされない。

### リネームルール

| 状況 | 結果 |
|---|---|
| 名前が未使用 | そのまま登録 |
| 既存・**action も完全一致** | スキップ（重複取り込みガード） |
| 既存・action が違う | `<名前>_2026-04-19` にリネーム |
| `_2026-04-19` もある | `_2`, `_3` と連番付与 |
| preset と同名 | 常にリネーム（preset 不可侵） |

### リネーム例

```
[受信] {"command":"grid","action":"新しいコード"}

既存の $grid は preset のまま → 新規は $grid_2026-04-19 として登録
```

## 4. バンドル取り込み時の3択ダイアログ

**キャンバス付きのバンドル**（`blackmirror.bundle`）を取り込む時だけ、確認ダイアログが出る：

```
◼ BUNDLE DETECTED
incoming: 12 object(s), 3 spell(s)
current canvas: 5 object(s)

[O] OVERWRITE — 現行を消して置き換え
[A] ADD — 現行に追加マージ
[C] CANCEL
```

| ボタン | 挙動 |
|---|---|
| **OVERWRITE** | 現キャンバスを全消去してバンドルの絵で置換。ビュー座標も復元 |
| **ADD** | 現キャンバスを残したまま bundle の絵を追加。ID 衝突は自動再採番 |
| **CANCEL** | 何もしない（モーダル外クリックでも可） |

魔法は自動リネームなので、どちらを選んでも衝突しない。

### どちらを選ぶか？

| 状況 | 推奨 |
|---|---|
| 他人のワークスペースを丸ごと開きたい | **OVERWRITE** |
| 空のボードに読み込みたい | どちらでも同じ（現キャンバス空のため） |
| 自分の作業に他人の絵を重ねたい | **ADD** |
| 間違えた | **CANCEL** |

## 5. ペーストでの実際の流れ

### X から魔法を拾う

```
1. 友人が投稿：
   「この魔法どうぞ→ {"command":"grain","action":"..."}」
2. JSON 部分だけをコピー
3. Black Mirror Board のターミナルにペースト
4. 自動登録
5. $ grain で実行
```

### チームの Slack から一括取り込み

```
1. チームの魔法ライブラリチャンネルにある JSON をコピー
2. 配列形式 [{...},{...}] でも OK
3. ペーストで一気に複数登録
```

---

## 6. D&D での実際の流れ

### Drive からダウンロードして取り込み

```
1. Drive から team_spells.json をダウンロード
2. ダウンロードフォルダからボードにドラッグ
3. 自動判別 → 複数魔法を一括登録
4. list で確認
```

### 複数マシンでの同期

```
Mac:
1. export data で全部バンドル化
2. iCloud Drive に保存

iPad:
3. Safari で Black Mirror Board を開く
4. Files アプリから JSON をドラッグ
5. OVERWRITE でワークスペース復元
```

## 7. 取り込んだ魔法の確認

### 一覧表示

```
list
```

→ 登録された全魔法が表示される（preset / user 区別付き）

### 内容確認

特定の魔法の中身を見たいとき：

```
share <名前>
```

→ クリップボードに JSON がコピーされるので、メモアプリ等に貼って中身を読める。

---

## 8. 取り込みを拒否したい場合

### 魔法を削除

```
forget <名前>
```

preset は削除不可。user 魔法のみ削除可能。

### リネームされた魔法を整理

```
forget grid_2026-04-19
```

日付付きの取り込み魔法をあとで整理するときに。

---

## 9. 取り込みの制限と注意

### ローカルファイルのみ

**URL から直接取り込む機能はない**。事前にダウンロードしてから D&D する必要がある。

### 画像込みバンドルのサイズ

画像が dataURL として埋め込まれているバンドルは数 MB になることも。モバイルだと遅い場合がある。

### 不正な JSON

JSON のシンタックスエラー、スキーマ不一致は受け付けられず、エラーログが出る：

```
[drop] "xxx" — unrecognized JSON format.
```

その場合は JSON の中身を確認。

## 10. 受信側のユーザー体験チェックリスト

自分が配布する側のとき、受信者がスムーズに取り込めるように：

- [ ] 配布前に自分のボードで動作確認した
- [ ] 魔法名はわかりやすいか
- [ ] action に CORS 制限のある外部 API 依存がないか
- [ ] `BM.create('rect')` 等の無効な type を使っていないか
- [ ] README や投稿文で使い方を説明しているか
- [ ] 配布形式が相手の環境に合うか（PC派 / モバイル派）
