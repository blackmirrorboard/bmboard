# 05. AI に魔法を作らせるガイド

> Claude / ChatGPT に「こんな魔法が欲しい」と頼むだけで JSON を生成してもらう方法。
> **このガイドをそのままコピペで AI に渡せる**ように設計されています。

---

## なぜ AI に作らせるのか

- JavaScript を書けなくても魔法が作れる
- 複雑な魔法も自然言語で依頼できる
- 失敗パターンを学習させれば確実性が上がる

ただし **AI は BM API の仕様を知らない** ので、**前提情報を必ず与える**必要があります。

---

## プロンプトテンプレート（コピペ用）

以下の内容を AI に渡せば、高確率で動く魔法 JSON が返ってきます。

````
あなたは Black Mirror Board の「魔法」を作るアシスタントです。
以下のルールと API 仕様を厳守して、JSON 形式で魔法を1つ作ってください。

## 出力形式（この形式以外で返さないこと）

{
  "command": "魔法の名前（英数字・ハイフン・アンダースコアのみ、30文字以内）",
  "action": "JavaScript コード（1行 or 複数行の文字列）"
}

## BM API（action の中で使える関数はこれらだけ）

参照・検索:
- BM.getSelected()              選択中のオブジェクト配列
- BM.all()                      全オブジェクト配列
- BM.getById(id)                ID で取得
- BM.find(fn)                   条件関数で検索

生成・変更:
- BM.create(type, props?)       新規作成（type は下記5種のみ）
- BM.update(obj|id, patch)      属性一括更新
- BM.translate(obj, dx, dy)     移動
- BM.setStroke(obj, color)      線色
- BM.setFill(obj, color)        塗り色
- BM.remove(obj)                削除
- BM.clear()                    全削除

状態:
- BM.viewCenter()               { x, y }
- BM.rand(min, max)             乱数
- BM.getMode()                  テーマ名
- BM.setMode(mode)              テーマ切替

描画・保存:
- BM.redraw()                   再描画（変更後は必ず呼ぶ）
- BM.save()                     Undo 履歴に保存
- BM.log(msg, cls?)             ターミナル出力（cls: 't-ok'/'t-err'/'t-dim'）

## BM.create で使える type は5種のみ

- circle    { cx, cy, rx, ry }
- square    { x, y, w, h }     ← 注意：'rect' は無い！
- triangle  { x, y, w, h }
- arrow     { x1, y1, x2, y2 }
- text      { x, y, text, fontSize }

## 禁止事項（これらは使えない・書くとエラー）

- BM.create('rect', ...)        ← 'square' を使うこと
- document.*                    ← DOM 直接操作は非推奨
- window.*                      ← 同上
- fetch() / XMLHttpRequest      ← 使えるが CORS に注意
- import / require              ← 使えない
- 外部ライブラリ                 ← BM API のみ
- async/await                   ← 使えない（action は同期関数として実行）

## action の書き方

- 変数は const / let を使う（var は避ける）
- 変更後は BM.redraw() を忘れずに
- 重要な変更は BM.save() で Undo 対応に
- 選択必須の魔法は、選択が無い場合に BM.log('select first', 't-err'); return; を入れる
- 実行完了時は BM.log('xxx done') で手応えを示す

（ここに作ってほしい魔法を自然言語で書く）
例：選択中の図形を画面中央に集める魔法
````

## 使用例1：AI に頼んでみる

**あなたの入力：**
```
リクエスト：選択中のオブジェクトを全て画面中央に集める魔法を作って
```

**AI の出力（期待されるもの）：**
```json
{
  "command": "center-all",
  "action": "const sel=BM.getSelected();if(!sel.length){BM.log('select first','t-err');return}const vc=BM.viewCenter();for(const o of sel){if(o.data.cx!==undefined){o.data.cx=vc.x;o.data.cy=vc.y}else if(o.data.x!==undefined){o.data.x=vc.x-(o.data.w||0)/2;o.data.y=vc.y-(o.data.h||0)/2}}BM.redraw();BM.save();BM.log('centered '+sel.length+' object(s)')"
}
```

**取り込み：**
このまま JSON をコピーしてターミナルにペースト → 自動登録。

## 使用例2：よくある失敗と対処

### 失敗パターン1：`rect` を使ってしまう

AI は JavaScript の慣習で `rect` を使いがち：

```
"action": "BM.create('rect', {x:0, y:0, w:100, h:100})"
```

→ **エラー**：`BM.create: unknown type "rect"`

**対処**：プロンプトで「rect は無い、square を使う」を強調する。

### 失敗パターン2：`BM.redraw()` を忘れる

データは変わっているのに画面が更新されない。

**対処**：プロンプトで「変更後は必ず BM.redraw()」を明示。

### 失敗パターン3：外部ライブラリを使おうとする

```
"action": "import lodash from 'lodash'; ..."
```

→ **動かない**。

**対処**：プロンプトで「BM API のみ使用可」を強調。

### 失敗パターン4：async/await を使う

```
"action": "const img = await fetch(...); ..."
```

→ action は同期関数として実行されるため、async は機能しない。

**対処**：Promise チェーン（.then）で書かせる。

## 魔法の依頼の仕方のコツ

### ✅ 良い依頼

```
選択中の円だけを赤く塗りつぶす魔法
```

→ 具体的、対象が明確、挙動が1つ

### ✅ もっと良い依頼

```
選択中の円だけを赤く塗りつぶして、円以外は無視する魔法。
選択が空なら "no circle selected" とエラー表示する。
```

→ エッジケース（選択なし）まで指定

### ❌ あいまいな依頼

```
円を赤くするやつ
```

→ 全円？選択中だけ？ 塗り？線だけ？

### ❌ BM API の範囲外の依頼

```
YouTube の動画を埋め込む魔法
```

→ 動画埋込は BM.create の対応外。できないことを依頼しないよう [[06_できることできないこと一覧|06. できる・できない]] を参照。

## 動作確認フロー

AI が作った JSON を取り込んだら、必ず **動作確認**：

```
1. JSON をターミナルにペースト
2. [OK] spell acquired — run: $<名前> が出るか確認
3. $ <名前> を実行
4. 意図通りに動くか確認
5. エラーが出たら → AI に返してエラー内容を伝える
```

### エラー時の AI への返し方

エラーが出たら、エラー文をそのままコピペして AI に戻す：

```
エラーが出ました。
[grain] runtime error: BM.create: unknown type "rect"

修正して再度 JSON を出力してください。
```

AI は多くの場合、これで正しく直します。

## 応用：AI と対話して魔法を育てる

1. 最初は簡単な魔法を頼む
2. 動作確認
3. 「次は選択なしの時にエラー表示する挙動を追加して」
4. 「次はランダム色を追加して」
5. …と段階的に機能追加

**動く状態を保ちながら育てる**のがコツ。一気に複雑な依頼をすると失敗しやすい。
