# 03. 魔法の基本・プリセット活用

> 「魔法」とは何か。最初から入っている5つの魔法を使いこなす。

---

## 魔法とは何か

Black Mirror Board で言う「魔法」とは、**ターミナルから呼び出せるカスタムコマンド**のこと。

### 通常コマンドとの違い

| | 通常コマンド | 魔法 |
|---|---|---|
| 例 | `save` / `load` / `dark` / `help` | `scatter` / `biwako-blue` / `fetch-img` |
| 定義元 | アプリ本体 | 個別の JS コード |
| 追加・削除 | 不可 | 自由（`register` / `forget`） |
| 共有 | 不可 | JSON でやりとり可能 |

つまり、**魔法 = JSON で表現できるコマンド**。これが Black Mirror Board の最大の特徴。

---

## プリセット魔法（最初から入っている5つ）

### 1. `$ scatter` — 散布

選択中のオブジェクトをランダムに散らす。

**使い方**

```
1. 複数のオブジェクトを選択（Shift+クリック or マーキードラッグ）
2. ターミナルに scatter → Enter
3. ランダムに散らばる
```

**用途**：構図の固定観念を崩すブレスト、ランダムレイアウトの試作。

---

### 2. `$ grid-3x3` — 秩序の召喚

ビュー中心に 3×3 の四角グリッドを配置。

**使い方**

```
ターミナルに grid-3x3 → Enter
```

**用途**：レイアウトの骨格作り、サムネのベース、モックアップの土台。

---

### 3. `$ biwako-blue` — 静寂の彩色

キャンバス上の全オブジェクトのストロークを青 `#0044CC` に染める。

**使い方**

```
ターミナルに biwako-blue → Enter
```

**用途**：単色ドラフトから色付き仕上げへの一括切替。気に入らなければ Cmd+Z で戻せる。

---

### 4. `$ fetch-img` — 画像召喚（v1.2 新仕様）

キーワードを元に画像を取ってきて配置する。

**使い方**

```
fetch-img                  → デフォルト "biwako" で1枚
fetch-img mountain         → "mountain" で1枚
fetch-img mountain 6       → 6枚・自動で3×2グリッド配置
fetch-img tokyo night 9    → "tokyo night" で9枚・3×3グリッド
```

**仕様**

- API：Lorem Flickr（無料・キー不要）
- 最大25枚
- カラー画像が返る（モノクロにしたければ `$ monoclo` で後がけ）
- グリッド配置は `cols = ceil(√n)` で自動計算

**用途**：ムードボード作成、参考画像コレクション、ブレスト中の視覚刺激。

---

### 5. `$ monoclo` — モノクロ化（v1.2 新規）

選択中の画像をモノクロに変換する。

**使い方**

```
1. 画像を選択（複数選択も可）
2. ターミナルに monoclo → Enter
3. モノクロになる
4. 気に入らなければ Cmd+Z
```

**仕様**

- 対象は **画像のみ**（シェイプ・ペン線には効かない）
- フィルタ：grayscale(100%) contrast(110%) brightness(95%)
- 変換結果はボードに永続化（リロード後も維持）

**用途**：カラー画像で取ってきたムードボードを一気にモノクロ統一したいとき。

---

### 6. `$ svg` — ベクター書き出し（v1.2）

選択オブジェクトを編集可能な SVG として保存する。Illustrator / Figma / Inkscape でそのまま開けて、各オブジェクトは独立したレイヤー/パスとして編集可能。

**使い方**

```
1. 書き出したいオブジェクトを選択（鉛筆線・テキスト・シェイプ・画像、混在可）
2. ターミナルに svg → Enter
3. BM_CANVAS_YYYYMMDD_HHMM.svg として保存される（iOS は共有シート）
4. 選択が空の場合はキャンバス全体を書き出し
```

**仕様**

- 対象: 7型すべて（circle / square / triangle / arrow / text / pen stroke / image）
- マッピング:
  - circle → `<ellipse>`
  - square → `<rect>`
  - triangle → `<polygon>`
  - arrow → `<line>` + 先端用 `<polygon>`
  - text → `<text dominant-baseline="hanging">`
  - pen stroke → `<path d="M ... Q ...">`（closed なら末尾 `Z` と fill）
  - image → `<image href="...">`（dataURL 優先、URL のみは外部依存）
- 回転: `<g transform="rotate(deg cx cy)">` でラップ
- 色: 現在のテーマ色（`resolveFill()` / `getStrokeColor()`）をそのまま書き出す
- viewBox: 対象オブジェクトのバウンディングボックス + 20px パディング
- ファイル名: `BM_CANVAS_YYYYMMDD_HHMM.svg`

**注意事項**

- **DARK モードから書き出すとストロークが淡色**になります。白背景のソフトで開くと見えないので、事前に `light` / `gray` モードに切り替えるか、閲覧側の背景を暗色にすること
- `fetch-img` で取得した画像のうち CORS で dataURL 化できなかったものは URL 参照のまま → 受信側がオフラインだと表示できない
- テキストのフォントは `Courier New, monospace` を指定。Illustrator / Figma で別フォントに差し替え可能

**用途**

- Illustrator / Figma への流し込み
- プレゼン資料・印刷物用の高精細ベクター素材
- ロゴ・アイコン系スケッチの最終仕上げ

## プリセットを組み合わせたワークフロー例

### ムードボード作成フロー

```
1. fetch-img ocean 9       → 海の写真9枚を3×3で配置
2. 気に入らない画像をクリック → Del で削除
3. 残った画像を全選択
4. monoclo                 → モノクロに統一
5. T キーで説明テキスト追加
6. export                  → 透過 PNG で書き出し
```

### レイアウト検討フロー

```
1. grid-3x3                → 3×3 骨格生成
2. 各セルを手動でカスタマイズ（ペン・図形・テキスト）
3. biwako-blue             → 全体を青く
4. cmd+Z で戻して色のバリエーション検討
```

---

## プリセットは削除できない

```
forget scatter
→ forget: "scatter" is a preset
```

preset は常に残る。自作魔法（`[ user ]` 表示）だけが `forget` で削除可能。

---

## プリセットの上書きは可能？

**不可能**。ペースト/D&D で同名の魔法が来た場合、preset は保護され、代わりに **リネームされた新規魔法として登録** される：

```
[preset] $scatter  （これは残る）
[ user ] $scatter_2026-04-19  （取り込んだもの）
```

自分の `register` で同名を登録しても、preset への上書きはできない仕様。
