# 07. 魔法の書き出しと共有

> 作った魔法・描いたキャンバスを他人に渡す／保存する方法の完全ガイド。

---

## 書き出しの手段

目的に応じて使い分け：

| 目的 | コマンド | 出力先 |
|---|---|---|
| 1つの魔法を SNS に貼る | `share <名前>` | クリップボード |
| 複数の魔法を選んでファイル化 | `export magic` | ファイル（.json） |
| ワークスペース全体を保存 | `export data` | ファイル（.json） |
| 絵を PNG で取り出す | `export` | ファイル（.png） |
| 絵をベクター SVG で取り出す | `$ svg` | ファイル（.svg） |

---

## 1. `share` — 1つの魔法をクリップボードに

### 使い方

```
share <魔法名>
```

### 例

```
share grain
```

→ クリップボードに以下の JSON がコピーされる：

```json
{
  "command": "grain",
  "action": "for(let i=0;i<120;i++){...}"
}
```

### 用途

- X（旧 Twitter）に貼って配布
- Discord の魔法共有チャンネルに投稿
- Slack でチームに共有
- メモアプリで保管

### 受け取り側

ターミナルにペーストするだけで自動登録される（Enter も不要）。

---

## 2. `export magic` — 魔法ピッカー（対話モーダル）

### 使い方

```
export magic
```

→ モーダルが開く：

```
◼ EXPORT SPELLS
書き出したい魔法にチェック → EXPORT SELECTED

[SELECT ALL] [USER ONLY] [CLEAR]              3 selected

☐ [preset] $biwako-blue
☐ [preset] $fetch-img
☐ [preset] $grid-3x3
☑ [ user ] $grain
☐ [preset] $monoclo
☐ [preset] $scatter
☑ [ user ] $random-circles
☑ [ user ] $my-scatter

                        [CANCEL] [EXPORT SELECTED]
```

### 操作

| ボタン | 効果 |
|---|---|
| **SELECT ALL** | 全部チェック |
| **USER ONLY** | preset を除外して自作魔法だけチェック（**基本これ**） |
| **CLEAR** | 全部解除 |
| **EXPORT SELECTED** | 選んだ魔法をファイル化 |
| **CANCEL** | 中止 |

### 出力ファイル

```
BM_PROJECT_20260419_1530_SPELLS.json
```

形式：

```json
{
  "format": "blackmirror.spells",
  "version": "1.2",
  "exportedAt": "2026-04-19T...",
  "commands": [
    { "command": "grain", "action": "..." },
    { "command": "random-circles", "action": "..." }
  ]
}
```

### 用途

- 友人に魔法セットを配る
- 自分のマシンを乗り換える前のバックアップ
- チーム内での魔法ライブラリ共有

---

## 3. `export data` — データバンドル（粒度調整）

### 使い方

```
export data
```

→ モーダルが開く：

```
◼ EXPORT DATA
書き出す内容を選択 → EXPORT BUNDLE

[✓] include canvas
    12 object(s) in current board

[✓] include user spells
    4 spell(s) registered by you

[ ] include preset spells
    5 preset(s) — 受信側が既に持っているので通常OFF

                        [CANCEL] [EXPORT BUNDLE]
```

### 自動フォーマット選択

| チェック状態 | 出力形式 | ファイル名 |
|---|---|---|
| canvas を含む | `blackmirror.bundle` | `*_BUNDLE.json` |
| canvas 無し（魔法のみ） | `blackmirror.spells` | `*_SPELLS.json` |

### 用途

- ワークスペース全体のバックアップ
- 別マシンへの引越し
- 他人と共同作業するためのベースファイル配布

---

## 4. `export` — 透過 PNG 書き出し

### 使い方

```
export
```

または `X` キー。

### 挙動

- **選択中のオブジェクトがあれば** → それだけを書き出し
- **何も選択していなければ** → 全オブジェクトを書き出し
- 背景透過 PNG として保存
- ブラウザの別名保存ダイアログが出る

### 用途

- Figma / Photoshop / Illustrator へのインポート
- プレゼン資料への貼り付け
- SNS への画像投稿

---

## 4.5 `$ svg` — ベクター SVG 書き出し（v1.2 新規）

### 使い方

```
svg
```

または `$ svg`。

### 挙動

- **選択中のオブジェクトがあれば** → それだけを書き出し
- **何も選択していなければ** → キャンバス全体を書き出し
- ファイル名: `BM_CANVAS_YYYYMMDD_HHMM.svg`
- iOS では共有シートが開く

### 対応オブジェクト → SVG 要素

| 型 | SVG 要素 |
|---|---|
| 円 (`circle`) | `<ellipse cx cy rx ry>` |
| 四角 (`square`) | `<rect x y width height>` |
| 三角 (`triangle`) | `<polygon points="...">` |
| 矢印 (`arrow`) | `<line>` + 先端 `<polygon>` |
| テキスト (`text`) | `<text dominant-baseline="hanging">` |
| 鉛筆線 (`stroke`) | `<path d="M ... Q ...">`（closed なら `Z` + fill） |
| 画像 (`image`) | `<image href="...">` |

- 回転は `<g transform="rotate(deg cx cy)">` でラップ
- 色は現在のテーマ色をそのまま直書き（resolveFill で解決）
- viewBox は対象オブジェクトのバウンディングボックス + 20px パディング

### 用途

- Illustrator / Figma / Inkscape でのプロ仕上げ
- 印刷物・名刺・ポスター用の高精細ベクター素材
- プレゼン資料への流し込み（解像度フリー）
- ロゴ・アイコンのドラフト → プロツールでの洗練

### 注意

- **DARK モードから書き出すとストロークが淡色になる**。白背景のソフトでは見えないので、`light` / `gray` に切り替えてから書き出すか、受け側の背景を暗くする
- `fetch-img` の画像で dataURL 化に失敗しているものは URL 参照のみ → オフラインや受信側環境で表示されない可能性
- テキストフォントは `Courier New, monospace`。Illustrator / Figma で好きなフォントに差し替え可能

### 使用例

```
1. S キーでセレクトツール
2. ドラッグで複数オブジェクトをマーキー選択
3. ターミナルに svg → Enter
4. BM_CANVAS_20260420_1930.svg がダウンロードされる
5. Illustrator で開く → 全オブジェクトが独立レイヤーとして編集可能
```

## 5. 非対話版のコマンド（一発書き出し）

対話モーダルを経由せずに書き出したい場合：

| コマンド | 挙動 |
|---|---|
| `export-spells` | 自作魔法すべてをファイル化 |
| `export-spells grain paint-red` | 指定した魔法だけ |
| `export-all` | キャンバス+全魔法（preset 込み） |

## 6. SNS での共有フロー

### X / Discord で1つの魔法を配布

```
1. share <魔法名>              → クリップボードに JSON
2. X / Discord に貼る
3. 受け取った人がペーストで取り込み
```

### 投稿テンプレート例

```
新しい魔法「grain」を作りました🔮
ボードに降らすとフィルムグレイン質感が出ます。

{"command":"grain","action":"for(let i=0;i<120;i++){...}"}

#blackmirror
```

## 7. ファイル共有フロー

### Google Drive / iCloud でチーム配布

```
1. export magic → SPELLS.json をダウンロード
2. Drive にアップロード
3. チームに URL を共有
4. 各自が DL → ボードに D&D で一括取り込み
```

### GitHub での公開

```
1. GitHub リポジトリに .json ファイルを置く
2. README に説明と JSON の中身を書く
3. 欲しい人は Raw URL から DL → D&D
```

将来的にコミュニティ用リポジトリができる予定。

## 8. 魔法の命名のコツ（配布を意識）

| ❌ あまり良くない | ✅ 良い命名 |
|---|---|
| `test1` | `grain-120` |
| `new` | `random-circles-5` |
| `mine` | `my-scatter-wider` |

**動作がわかる短い英単語**がベスト。他人が `list` で見たときに何するか察せる名前を。

---

## 9. 書き出し時の注意

### CORS で壊れている画像の扱い

`fetch-img` で取得した画像のうち、CORS 制限で dataURL 化に失敗したものは **エクスポートバンドルに含まれない**（URL 参照のみ）。相手のブラウザでも表示されるかは Lorem Flickr が生きているかに依存。

→ 確実に残したい画像は `I` キーで PC からアップロードすると dataURL 化される。

### ファイルサイズの目安

| 内容 | サイズ |
|---|---|
| 魔法1つ（action 500字程度） | 〜1 KB |
| 魔法10個 | 5〜10 KB |
| キャンバス（図形100個） | 20〜50 KB |
| キャンバス（画像5枚込み） | 500 KB〜数 MB |

画像込みだと重くなるので注意。

## 10. iOS での書き出し

iPhone/iPad では `export-all` `export-spells` `export data` すべて **共有シート** が自動で開く：

- AirDrop で Mac に転送
- iCloud Drive に保存
- メール／メッセージで送信
- Drive / Dropbox にアップロード
